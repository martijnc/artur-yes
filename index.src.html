Introduction {#intro} ============ Content Security Policy [[CSP]] is difficult to use in some cases, as the syntax is complicated, the overriding rules are confused, and the feature itself serves several different purposes. This document proposes a subset of CSP focused strictly on XSS protection, inspired in large part by Artur, et al.'s work on `'strict-dynamic'`. The goal is to provide the same guarantees as a verbose and complicated CSP, in a pithy and comprehensible package. Framework {#framework} ========= The `Artur` HTTP Response Header {#header} -------------------------------- The `Artur` HTTP response header field is the delivery mechanism for the XSS protections this document offers. The header's value is represented by the following ABNF [[!RFC5234]]: 
  Artur = json-field-value

    Artur: { "nonce": "abcdefg", "report": "group1" }
A server MAY send different `Artur` header field values with different representations of the same resource. The headerâ€™s value is interpreted as an array of JSON objects, as described in Section 4 of [[!HTTP-JFV]]. Algorithms {#algorithms} ---------- When executing [[csp3#set-response-csp-list]], run the following steps before the current step 4: 4. Let |list| be the result of executing the algorithm defined in Section 4 of [[HTTP-JFV]] on the value of |response|'s `Artur` header. If that algorithm does not result in error, then, for each |item| in |list|: 1. If |item| does not have a `nonce` member, or |item|'s `nonce` member's value does not match the nonce-source grammar, skip the remaining substeps, and move to the next |item|. 2. Let |policy| be a new policy with an empty directive set, and a disposition of "`enforce`". 3. Add a directive to |policy| whose name is "`script-src`", and whose value is the value of |item|'s `nonce` member. 4. If |item| has a `report` member whose value matches the `token` grammar [[RFC7230]]: 1. Add a directive to |policy| whose name is "`report-to`", and whose value is the value of |item|'s `report` member. 5. Append |policy| to |policies|.
